# Railway 로그 확인 가이드

## Railway 대시보드에서 로그 확인하는 방법

### 방법 1: 서비스에서 직접 확인 (가장 간단)

1. **Railway 웹사이트 접속**: https://railway.app
2. **프로젝트 선택**: 
   - 대시보드에서 프로젝트 클릭
   - 또는 왼쪽 사이드바에서 프로젝트 선택
3. **서비스 선택**:
   - 프로젝트 대시보드에서 `ncssearch-backend` 서비스 카드 클릭
   - 또는 서비스 목록에서 `ncssearch-backend` 클릭
4. **로그 확인**:
   - 서비스 페이지 상단에 탭 메뉴가 있습니다:
     - **Deployments** (배포 이력)
     - **Variables** (환경 변수)
     - **Metrics** (리소스 사용량)
     - **Logs** (로그) ← 여기!
   - **"Logs" 탭 클릭**

### 방법 2: 배포 이력에서 확인

1. **Railway 대시보드 접속**
2. **프로젝트 선택**
3. **`ncssearch-backend` 서비스 클릭**
4. **"Deployments" 탭 클릭**
5. **최신 배포 클릭** (가장 위에 있는 배포)
6. **배포 상세 화면에서 로그 확인**:
   - 배포 단계별 로그가 표시됩니다
   - "Build", "Deploy" 등의 단계를 클릭하여 로그 확인

### 방법 3: 서비스 카드에서 바로 확인

프로젝트 대시보드에서:
- `ncssearch-backend` 서비스 카드에 마우스 오버
- 또는 서비스 카드 우측 상단의 메뉴 아이콘 클릭
- "View Logs" 옵션 선택 (있는 경우)

## 로그에서 확인할 내용

### 정상인 경우:

```
✅ PostgreSQL 데이터베이스 연결 성공
📊 데이터베이스 모드로 API 서버 실행 중
🚀 API 서버가 http://0.0.0.0:3000 에서 실행 중입니다.
```

### 문제가 있는 경우:

```
⚠️ 데이터베이스 연결 실패. Mock 데이터 모드로 동작합니다.
   환경 변수를 확인하세요: DB_HOST, DB_NAME, DB_USER, DB_PASSWORD
```

또는:

```
❌ PostgreSQL 연결 오류: ...
```

## 찾기 어려운 경우

### 1. 사이드바 메뉴 확인

Railway 대시보드 왼쪽 사이드바에서:
- 프로젝트 목록 확인
- 프로젝트 클릭 → 서비스 목록 확인

### 2. 검색 기능 사용

Railway 대시보드 상단에 검색창이 있다면:
- "ncssearch-backend" 검색
- 서비스 선택

### 3. URL로 직접 접근

서비스 URL 형식:
```
https://railway.app/project/[프로젝트ID]/service/[서비스ID]
```

## 스크린샷으로 설명

### Step 1: 프로젝트 대시보드
- 여러 서비스 카드가 표시됨
- `ncssearch-backend` 카드 찾기

### Step 2: 서비스 클릭
- 서비스 카드를 클릭하면 상세 페이지로 이동

### Step 3: 탭 메뉴 확인
- 페이지 상단에 탭 메뉴:
  - Deployments
  - Variables  
  - Metrics
  - **Logs** ← 이 탭 클릭!

### Step 4: 로그 확인
- 실시간 로그가 스크롤되며 표시됨
- 최신 로그가 하단에 표시됨

---

## 대안: 배포 이력에서 확인

Logs 탭을 찾기 어렵다면:

1. **"Deployments" 탭 클릭**
2. **최신 배포 클릭** (가장 위에 있는 것)
3. **배포 단계별 로그 확인**:
   - "Initialization" 단계
   - "Build" 단계
   - "Deploy" 단계
   - 각 단계를 클릭하여 상세 로그 확인

---

## 문제 해결

### 로그가 보이지 않는 경우:

1. **서비스가 실행 중인지 확인**:
   - 서비스 상태가 "Online"인지 확인
   - "Offline"이면 서비스를 시작해야 함

2. **배포가 완료되었는지 확인**:
   - "Deployments" 탭에서 배포 상태 확인
   - "Building" 또는 "Deploying" 중이면 완료될 때까지 대기

3. **브라우저 새로고침**:
   - F5 또는 Ctrl+R로 페이지 새로고침
