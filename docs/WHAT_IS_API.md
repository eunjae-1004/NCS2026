# 실제 API란 무엇인가?

## 현재 상황

현재 프로젝트는 **프론트엔드만** 구현되어 있고, **백엔드 API 서버는 없습니다**.

대신 **Mock 데이터**를 사용하여 동작하고 있습니다.

## Mock 데이터 vs 실제 API

### Mock 데이터 (현재 상태)
- 프론트엔드 코드 내에 하드코딩된 데이터
- `src/services/mockData.ts`에 저장된 가짜 데이터
- 실제 데이터베이스 없이 동작
- 개발/테스트용

### 실제 API (목표)
- 별도의 백엔드 서버에서 제공하는 HTTP 엔드포인트
- 실제 데이터베이스와 연동
- 데이터를 영구적으로 저장하고 관리
- 프로덕션 환경에서 사용

## 실제 API 서버 구조

```
프론트엔드 (React)          백엔드 (Node.js/Express)
     │                            │
     │  HTTP 요청                  │
     ├───────────────────────────>│
     │  GET /api/ability-units    │
     │                            │
     │                            │  데이터베이스 조회
     │                            ├─────────────────>│
     │                            │                  │
     │                            │  데이터 반환     │
     │                            │<─────────────────│
     │                            │
     │  HTTP 응답                 │
     │<───────────────────────────┤
     │  { success: true, data: [...] }
     │
```

## 실제 API 서버 예제

프로젝트에 간단한 예제 API 서버를 추가했습니다:

- **위치**: `server/` 폴더
- **기술**: Node.js + Express
- **포트**: 3000

### 실행 방법

1. **서버 폴더로 이동**:
```bash
cd server
```

2. **의존성 설치**:
```bash
npm install
```

3. **서버 실행**:
```bash
npm run dev
```

4. **프론트엔드 설정 변경**:
프로젝트 루트에 `.env` 파일 생성:
```env
VITE_API_BASE_URL=http://localhost:3000/api
VITE_USE_MOCK_DATA=false
```

5. **프론트엔드 재시작**

## 실제 API의 장점

1. **데이터 영구 저장**: 데이터베이스에 저장되어 서버 재시작 후에도 유지
2. **실제 데이터 관리**: 실제 NCS 데이터를 관리하고 업데이트 가능
3. **사용자별 데이터**: 각 사용자의 선택 이력을 개별적으로 저장
4. **확장성**: 여러 사용자가 동시에 사용 가능
5. **보안**: 인증/인가 기능 추가 가능

## 다음 단계

1. ✅ 예제 API 서버 실행 (`server/` 폴더)
2. 실제 데이터베이스 연동 (PostgreSQL, MySQL 등)
3. 실제 NCS 데이터 로드
4. 인증/인가 시스템 추가
5. 프로덕션 배포

## 요약

- **현재**: Mock 데이터 사용 (프론트엔드만)
- **실제 API**: 백엔드 서버에서 제공하는 HTTP 엔드포인트
- **예제 서버**: `server/` 폴더에 제공됨
- **사용 방법**: `.env` 파일에서 `VITE_USE_MOCK_DATA=false` 설정
