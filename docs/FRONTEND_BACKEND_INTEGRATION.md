# 프론트엔드-백엔드 연동 가이드

## ✅ 완료된 작업

1. ✅ 데이터베이스 생성 및 데이터 import
2. ✅ 백엔드 API 서버 실행 중 (`http://localhost:3000`)
3. ✅ 데이터베이스 연결 확인
4. ✅ 환경 변수 파일 생성 (`.env`)

## 🔄 다음 단계

### 1. 프론트엔드 재시작

환경 변수를 적용하기 위해 프론트엔드를 재시작해야 합니다:

```bash
# 현재 실행 중인 프론트엔드 서버 중지 (Ctrl+C)
# 그 다음 다시 시작
npm run dev
```

### 2. 실제 API 테스트

프론트엔드가 재시작되면:

1. **홈 페이지에서 검색 테스트**
   - 키워드 검색: "품질", "관리" 등
   - 실제 데이터베이스에서 조회한 결과가 표시됩니다

2. **상세 페이지 확인**
   - 능력단위 상세 페이지에서:
     - ✅ KSA (지식/기술/태도) 데이터 표시
     - ✅ 수행준거 표시
     - ✅ 능력단위 요소 표시

3. **기관 선택 확인**
   - 로그인 페이지에서 기관 목록이 실제 DB에서 조회됩니다

## 🐛 문제 해결

### 프론트엔드에서 API 호출 실패

**확인 사항:**
1. 백엔드 서버가 실행 중인지 확인 (`http://localhost:3000`)
2. `.env` 파일이 프로젝트 루트에 있는지 확인
3. 브라우저 콘솔의 오류 메시지 확인
4. CORS 오류인지 확인 (이미 설정되어 있음)

**해결 방법:**
```bash
# 백엔드 서버 재시작
cd server
npm run dev
```

### Mock 데이터가 여전히 표시됨

**확인 사항:**
1. `.env` 파일의 `VITE_USE_MOCK_DATA=false` 확인
2. 프론트엔드 서버를 재시작했는지 확인
3. 브라우저 캐시 삭제 (Ctrl+Shift+R)

### API 응답이 느림

**원인:**
- 데이터베이스 쿼리 최적화 필요
- 대량의 데이터 조회

**해결 방법:**
- 페이지네이션 추가
- 인덱스 확인
- 쿼리 최적화
