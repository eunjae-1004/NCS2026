# 오류 발생 원인 설명

## 발생한 오류

여러 페이지 파일에서 **"Missing semicolon"** 에러가 발생했습니다.

- `RecommendationPage.tsx` - 12번째 줄
- `AbilityDetailPage.tsx` - 10번째 줄  
- `SearchResultsPage.tsx` - 10번째 줄

## 원인

### 문제가 된 코드 구조

```typescript
// ✅ 올바른 import 문
import { useState } from 'react'
import { useNavigate } from 'react-router-dom'
// ... 다른 import들

// ❌ 문제: import 문 다음에 바로 객체/배열이 시작됨
  {
    id: '1',
    code: 'NCS-001',
    // ...
  }
```

### 왜 에러가 발생했나?

1. **문법 오류**: TypeScript/JavaScript에서 import 문 다음에 바로 객체 리터럴이나 배열이 올 수 없습니다.
   - import 문 다음에는 변수 선언, 함수 선언, 클래스 선언 등이 와야 합니다.
   - 객체나 배열은 변수에 할당되거나 함수의 일부여야 합니다.

2. **잘못된 코드 구조**:
   ```typescript
   // ❌ 잘못된 예시
   import ... from ...
   {  // 이렇게 바로 시작하면 안 됨
     id: '1'
   }
   
   // ✅ 올바른 예시
   import ... from ...
   const data = {  // 변수 선언 필요
     id: '1'
   }
   ```

3. **원래 의도**: 이 코드들은 원래 Mock 데이터 배열이었을 것입니다:
   ```typescript
   const mockRecommendations = [
     {
       id: '1',
       // ...
     }
   ]
   ```
   하지만 변수 선언 부분(`const mockRecommendations =`)이 삭제되면서 배열만 남게 되었습니다.

## 왜 이런 일이 발생했나?

### 추정되는 과정

1. **초기 개발 단계**: 각 페이지 파일에 Mock 데이터가 하드코딩되어 있었습니다.
   ```typescript
   const mockAbilityUnits = [
     { id: '1', ... },
     { id: '2', ... }
   ]
   ```

2. **API 서비스 전환**: Mock 데이터를 제거하고 API 서비스를 사용하도록 변경하는 과정에서:
   - Mock 데이터 배열을 제거하려고 했지만
   - 변수 선언 부분만 삭제되고
   - 배열의 내용 부분이 남아있게 되었습니다.

3. **결과**: import 문 다음에 바로 객체/배열이 오는 잘못된 코드 구조가 되었습니다.

## 해결 방법

### 수정 전
```typescript
import ErrorMessage from '../components/ErrorMessage'
  {  // ❌ 변수 선언 없이 바로 객체 시작
    id: '1',
    code: 'NCS-001',
    // ...
  }
```

### 수정 후
```typescript
import ErrorMessage from '../components/ErrorMessage'

export default function RecommendationPage() {  // ✅ 함수 선언으로 시작
  // ...
}
```

## 교훈

1. **코드 리팩토링 시 주의**: 대량의 코드를 삭제할 때는 전체 구조를 확인해야 합니다.
2. **린터 활용**: TypeScript/ESLint가 이런 오류를 잡아주지만, 때로는 놓칠 수 있습니다.
3. **단계별 수정**: 한 번에 많은 파일을 수정하기보다는 하나씩 확인하며 수정하는 것이 안전합니다.

✅ 모든 파일 수정 완료
- `RecommendationPage.tsx` - Mock 데이터 제거 완료
- `AbilityDetailPage.tsx` - Mock 데이터 제거 완료
- `SearchResultsPage.tsx` - Mock 데이터 제거 완료

✅ 모든 파일이 올바른 구조로 변경됨
- import 문 → 함수/컴포넌트 선언 순서로 정리

## 예방 방법

1. **코드 삭제 시**: 변수 선언부터 끝까지 전체를 확인
2. **리팩토링 후**: 항상 컴파일 확인 (`npm run build` 또는 `npm run dev`)
3. **린터 사용**: `npm run lint`로 코드 검사
